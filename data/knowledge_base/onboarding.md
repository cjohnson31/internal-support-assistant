# Engineering Onboarding

## Week 1 Checklist

### Day 1
- [ ] Get laptop from IT (bring your offer letter)
- [ ] Set up SSO account at https://sso.company.com/setup with your company email
- [ ] Enable MFA on your SSO account (required before any system access)
- [ ] Connect to office Wi-Fi (`CorpSecure`) or install VPN (see VPN guide)
- [ ] Join Slack — your manager will invite you to team channels

### Day 2-3
- [ ] Access Atlas at https://atlas.internal.company.com (SSO login)
- [ ] Request **Developer** role via Access Request (your manager will approve)
- [ ] Set up your local development environment (see Dev Environment guide)
- [ ] Clone the team's repos from GitHub Enterprise

### Day 4-5
- [ ] Complete security awareness training at https://training.internal.company.com
- [ ] Familiarize yourself with the team's services in the Atlas service catalog
- [ ] Set up your first deployment pipeline for a test service
- [ ] Shadow the current on-call engineer for your team

## Common First-Week Issues

- **"I can't log into Atlas"**: Make sure MFA is set up on your SSO account first
- **"Access request is pending for days"**: Ping your manager — they may have missed the notification
- **"I can't clone repos"**: You need to add your SSH key to GitHub Enterprise (Settings → SSH Keys)
